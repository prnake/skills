# debug-languages

This skill helps the model more carefully check its code during single-round testing. Its main purpose is to ensure that the code is runnable and that the bug has been fixed before delivering the result.

The skill is written by K2.5 with simple prompt, no special design for SanityHarness. ( https://www.kimi.com/share/19c674bd-7f22-8d3e-8000-0000219c8ee7 )

Used in GLM-5 submission in SanityHarness, 61.5% -> 76.9%.
